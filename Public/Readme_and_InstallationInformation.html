Introduction

SAPUI5 is a state-of-the-art client-side rendering framework that:
  • ... provides RIA-style AJAX features
  • ... supports the SAP product standards
  • ... offers an extensibility concept
  • ... allows theming and customer branding
  • ... comes with a rich set of controls for building desktop as well as mobile applications
  • ... introduces controls that work on all form factors
  • ... provides a lightweight model-view-controller (MVC) concept, componentization and data binding
  • ... has out-of-the-box support for the OData protocol
  • ... implements advanced caching and resource handling mechanisms
  • ... allows on-the-fly creation of new controls
  • ... provides an Eclipse based design environment for development of SAPUI5 applications
  • ... is release independent with very few dependencies to other components

SAPUI5 is based on HTML5 and CSS3, so it should run on all modern browsers. Currently supported are Internet Explorer 9 (and - with some visual degradations IE8), the latest versions of Chrome, Firefox and (on Mac OS) Safari

What's new in version 1.12?

These are the highlights of SAPUI5's new evaluation version as opposed to the 1.8 version:
  • Components: One can bundle UI5 related objects like modules, views, controllers, etc. in components
  • New view type: Declarative HTML, this is also included in SAPUI5 Eclipse tools
  • New desktop controls like
    • Form, ResponsiveLayout, Autocomplete, Collection Inspector, FormattedTextView
    • Notetaker, DateRangeSlider, SplitButton
  • Best practices mobile demo application
  • All mobile (sap.m) controls which are not experimental support Blackberry 10
  • New mobile controls like
    • SplitApp, PullToRefresh, MessageToast, MessageBox, ActionSheet, InputDateTime
  • New experimental mobile controls like
    • TileContainer, TabularList, Shell, Accept/Reject Switch, GrowingList, IconTabBar, ActionSheet
    • DateTimeInput, Link, ObjectHeader, TextArea, URLHelper
    • Calendar, ProgressIndicator, TabContainer
  • New mobile features
    • Configurable/themable background images for App, SplitApp, Shell (experimental)
    • Swipe-for-action in Lists
    • Mobile library comes with new experimental Blue Crystal theme
    • Mobile library with Blue Crystal theme runs on smart phones, tablets and desktops. This is only an experimental feature
    • Charts in sap.viz library
      • Basic touch support
      • New charts: Dual Bar, Dual Line, Dual Combination, Dual Column, Dual Stacked Bar
    • New Icon fonts / pool
    • sap.ui.Device: API for device and feature detection and media query support
    • Calculated fields in databinding, i.e. bind control properties to more than one data property

    SAPUI5 Installation

    To use the archives you have downloaded, follow the installation information.

    The installation section here is set up according to the various file packages sapui5-sdk-static.zip, sapui5-sdk.war, sapui5-static.zip, sapui5.war, and getting-started-sample.war as they are provided with the downloaded archive for using SAPUI5.

    "SAPUI5 SDK Static": sapui5-sdk-static.zip

    Extract the sapui5-sdk-static.zip file to the web server that is available at your site. Under Developer Guide -> General Information -> Release Notes you can find some information with respect to what has changed in the current SAPUI5 version 1.12.1 compared to other older SAPUI5 versions. This is a short description of the most important content that is provided:

    /resources:

    Provides the JavaScript framework and control resource files required for setup, development, and debugging when working with the SAPUI5 user interface technology. The folder for example also contains the sap-ui-core-all.js file which is used in the sample included at the bottom of the page.

    /test-resources:

    The test resources are the basis of the included test suite which is an add-on to the SAPUI5 feature set and which allows you to exercise lively on the control examples, and to reuse examples, or parts of them, where required.

    index.html:

    Informs about the demo kit and provides basic information of the platform. The file is also preconfigured to be used as starting point for the whole bundle.

    To install the sapui5-sdk-static.zip file on an Apache HTTPD server, the steps are as following:

    1. Download the Apache HTTPD server.
    2. Go into the Apache HTTPD server directory on your computer.
    3. Go into the /htdocs folder.
    4. Extract the zip file into folder /sapui5.
    5. Generally, you would use port 8080 to let the demo kit run in the browser.

    To install the sapui5-sdk-static.zip file on Microsoft IIS, proceed steps as following:

    1. Install Microsoft IIS by using the Windows Program Manager. Activate the Microsoft Internet Information Services.
    2. Open the Microsoft IIS /inetpub directory.
    3. Open the Microsoft IIS /wwwroot folder.
    4. Extract the zip file to a folder /sapui5.
    5. For typical installations you would use port 80 to let the application run.

    "SAPUI5 SDK": sapui5-sdk.war

    The demo kit web application archive file contains the same application as the static version described above. You would use this demo kit package file alternatively to the sapui5-sdk-static.zip file for direct deployment on a Java Web Container. As an additional feature a search is available with this version as opposed to the static version. You directly deploy the current package file to the Java Web Container which is available at your site, for example Apache Tomcat; we recommend the newest 6.x version. You do not have to extract the web archive file before.

    If you wanted to deploy the .war file to Apache Tomcat for example, you would follow these steps:

    1. Stop the Tomcat server if it has been started and is running.
    2. Delete existing deployments, if there are any. The relevant folder for deployments is TOMCAT_HOME/webapps.
    3. Copy the sapui5-sdk.war file to the /webapps directory.
    4. Start the server again.
    5. You do not have to adapt the server.xml file, this is done automatically.

    SAPUI5 runtime static : sapui5-static.zip

    The SAPUI5 static bundle consists of static web content only: HTML, JavaScript files, style sheets, and images. It does not include any active components like Servlets which would for example require a running Java web container. The SAPUI5 static bundle contains the SAPUI5 core library, and a set of basic UI libraries.

    Using a common standard web server plus the content of the static SAPUI5 runtime .zip file allows you to test and try and find out the SAPUI5 features and functions. The sapui5-static.zip file contains analogous to the demo kit zip file resources and test resources, as well as an index.html for your convenience.

    For the installation of this static file version, follow the steps as they are described for sapui5-sdk-static.zip.

    SAPUI5 runtime: sapui5.war

    This archive can be used for direct deployment on a Java Web Container like Apache Tomcat. It contains the same client side runtime parts as the static runtime version. However, it also includes some optional Java server components which optimize the resource loading mechanisms if present. An example are the language dependent resource bundles. If a required language bundle doesn't exist, the static version without the server component will have to issue several requests before a fallback file is found. The .war version will resolve this on the server within the first request.

    For information on how to proceed with the runtime package, check the information which is available for the SAPUI5 SDK web archive.

    SAPUI5 runtime for mobile hybrid web containers: sapui5-mobile-static.zip

    SAPUI5 Mobile apps can be built either as web applications which are loaded from a certain URL, or as so-called "Hybrid Apps" which have a native app wrapper, but the entire user interface consists of a HTML viewer where the normal SAPUI5 content is displayed.

    Both have advantages and disadvantages. Hybrid apps can be published in App Stores, the whole application code and the UI5 library files can also be embedded into this Hybrid container, so these files are rather installed once than downloaded on every application start. On the other hand this means that the total library size matters because every user has to install everything. In normal web applications the library deployed on the server may be huge, but only the needed parts are downloaded at runtime.

    For these hybrid apps, the static package of SAPUI5 Mobile can be used to include all needed SAPUI5 resources, but still have a relatively small total library size because all test pages etc. are removed from this special package. The file is sapui5-mobile-static.zip.

    It contains the debug version of all Javascript files as well as the optimized and minified one (which are used by default) so it can be used for productive as well as debug purposes.

    To use this package e.g. in Sybase's Hybrid Web Container or Adobe's PhoneGap, it just needs to be unzipped inside the respective resource location of the app development project. The Hybrid Web Container/PhoneGap build will then include the files and make them available at runtime.

    Optimizing the package size: Delete unneeded files

    While the above file is already pretty small (and small enough for inclusion in hybrid apps), the size can be further reduced and optimized for specific applications by deleting unneeded files:

    • In /resources you can delete either sap-ui-core-nojQuery.js if you reference UI5 normally, or sap-ui-core.js if you reference the nojQuery version and reference jQuery separately (e.g. because you run in a Sybase Hybrid Container).
    • In /resources/sap you can delete the entire makit folder when you do not need the makit charts.
    • In /resources/sap/ui/thirdparty you can delete the entire jquery and jqueryui folders when you use the normal sap-ui-core.js file for bootstrap (they may only be needed when using the sap-ui-core-nojQuery.js script, and even then you may have another copy of jQuery available, so that folder can still be deleted).
    • In /resources/sap/ui/core/themes and in /resources/sap/m/themes you can delete the "base" and the "sap_bluecrystal" folder (unless you are using this experimental theme, then you can delete "base" and "sap_mvi")

    Of all JavaScript files there is an optimized and a "-dbg" version of each file, so when deleting the above files make sure you always delete both versions of them! If you do not want to allow easy debugging and you want to go for minimum installation size, also ALL *-dbg.js files can be deleted from the "-opt-static" package.

    Further files can be deleted, but the size reduction is limited and finding out what is not required gets increasingly difficult.

    Sample application: getting-started-sample.war

    Importing the content of the getting-started-sample file, which you have extracted before or deployed directly to your web container into an Eclipse IDE (for prerequisites and IDE version, check the prerequisites for the SAPUI5 Application Development Tools), you get a sophisticated SAPUI5 Application example which you can adapt for your purposes. The SAPUI5 Application Project is preconfigured in a way that you can directly let it run on your web server; enhance it with the SAPUI5 controls that fit for your application needs. The getting-started-sample file provides pre-thought steps which are already implemented for your convenience. For example, you can directly use the library packages as they are available in WebContent -> WEB-INF -> lib. The programming model foresees that it is sufficient to have the Java archive files available in the SAPUI5 Application Project, it is not necessary to extract these before using them.

    For just having a glance what a SAPUI5 Application looks like in the browser, you also could deploy it to your Web container only, and let it run via http://localhost:8080/getting-started-sample.

    Content Tools Update Site

    The overall structure of the tools update site folder is as following:

    /features:

    Contains the com.sap.ide.ui5.* and com.sap.ui5.* Java archives with runtime, controls and tools build files.

    /plugins:

    Contains the packages which provide the controls for UI design, as well as for example OSGI-enabling files for SAPUI5.

    artifacts.jar:

    In artifacts.xml, the artifact repository for the SDK with the required parameters is defined.

    content.jar:

    In the content.xml file, the metadata repository for the SDK with the required parameters is defined.

    Installation SAPUI5 Tools

    Only a few installation steps are required to get the full functionality of the features and functions. Note that when you are installing the SAPUI5 Tools via the tools update site available with this download, you also automatically get the SAPUI5 Core Runtime via the installable units.

    1. Extract the HTML5Evaluation_complete.zip to any server location.
    2. Start the Eclipse environment. To install the features and functions, choose Help -> Install New Software...
    3. Click Add ... (next to the Work with: input field).
    4. Click Local ... (next to the Name: input field).
    5. Navigate to the local update site location, and choose it as update source. Mark all plugins and features that shall be installed. We recommend, for SAPUI5 Application Development purposes, to generally choose all installation options that are available. Keep the Contact all update sites during install to find required software checkbox at the bottom of this dialog selected. Finish the installation process.
    6. Restart Eclipse.
    7. To validate your installation, try to create a new SAPUI5 Application Project via Eclipse menu File -> New -> Other.

    Quickstart: An easy SAPUI5 application

    In the folder of the downloaded runtime, open the index.html page for first instructions

    Just paste the following into a new HTML file on your web server with the SAPUI5 runtime.

    <!DOCTYPE html>
<html><head>
    <meta http-equiv='X-UA-Compatible' content='IE=edge' />
    <title>Hello World</title>

    <script id='sap-ui-bootstrap' type='text/javascript'
        src='/sapui5/resources/sap-ui-core-all.js'
        data-sap-ui-theme='sap_goldreflection'
        data-sap-ui-libs='sap.ui.commons'></script> 

<script>
    var btn = new sap.ui.commons.Button({
        text:'Hello World',
        press:function(){alert('Hello!')}
    });
    btn.placeAt('content');
</script>

</head>
<body class='sapUiBody'>
    <div id='content'></div>
</body>
</html>
